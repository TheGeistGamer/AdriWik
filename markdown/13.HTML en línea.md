# La sintaxis de Markdown

La sintaxis de Markdown tiene único propósito: servir como formato para la escritura web.

Markdowon no reemplaza a HTML, ni se le parece en absoluto. Su sintaxis es muy reducida y corresponde únicamente a un subconjunto muy pequeño de etiquetas HTML. La idea no es crear una sitaxis que facilite la inserción de etiquetas HTML. EN mi opinión, insertar etiquetas HTML ya es sencillo. El objetivo de Markdown es facilitar la lectura, la escritura y la edición de textos. HTML es un formato de publicación; Markdown es un formato de escritura. Pro lo tanto, la sintaxis de formato de Markdown solo aborda cuestiones que pueden expresarse en texto plano.

Para cualquier marcado que no esté cubierto por la sintaxis de Markdown, simplemente use HTML. No es necesario anteponerle ni delimitarlo para indicar que está cambiando de Markdown a HTML; solo use las etiquetas.

Las únicas restricciones son que los elementos HTML de nivel de bloque (por ejemplo: div, table, pre, p etc...) deben estar seprados del contenido cicundante por líneas en blanco, y las etiquetas de inicio y fin del bloque no deben tener sangría con tabulaciones ni espacios.Markdown es lo suficientemente inteligente como para no añadir etiquetas (p) adicionales (no deseadas) alrededor de las etiquetas HTML de nivel de bloque.

## Alerta

Alias: `no-inline-html`
Regla: `MD033`
`Markdownlint` tiene una regla (MD033) que *bloquea* HTML inline porque Markdown tiene su propia sintaxis para muchas cosas (incluyendo tablas)

<table>
    <tr>
        <td>Foo</td>
    </tr>
</table>

Tenga en cuenta que la sintaxis de formato Markdown no se preocesa dentro de las etiquetas HTML de nivel de bloque. Por ejemplo, no puede usar *enfasis* al estilo Markdown dentor de un bloque HTML

| Nombre | Edad |
|--------|------|
| Juan   | 25   |
| Ana    | 30   |

### 🔧 Desactivar la regla (si necesitas HTML)

```json
{
  "MD033": false
}
```

### Solo en una parte del archivo

```html
<!-- markdownlint-disable MD033 -->
<table>
  <tr>
    <td>Hola</td>
  </tr>
</table>
<!-- markdownlint-enable MD033 -->
```

### ⚠️ Ignorar solo una línea

```html
<!-- markdownlint-disable-next-line MD033 -->
<table><tr><td>Hola</td></tr></table>
```

Las etiquetas HTML de nivel de span (Ejemplo: span, cite o del) se pueden usar en cualquier parate de un párrafo, elemento de lista o encabezado de Markdown. Si lo desea, incluso puede usar etiquetas HTML en lugar del formato de Markdown; por ejemplo, si prefieres usar las etiquetas HTML "a" o "img" en lugar de la sintaxis de enlace o imagen de Markdown, puede hacerlo sin problema

A diferencia de las etiquetas HTML de nivel de bloque, la sintaxis de Markdown se procesa dentro de las etiquetas de nivel de span.
