# Documentación

Markdown es una opción natural para la documentación técnica. Empresas como GitHub están cambiando cada vez más a Markdown para su documentación.

- Read the Docs puede generar un sitio web de documentación a partir de sus archivos Markdown de código abierto. Simplemente conecte su repositorio de GitHub a su servicio y presione: leer los docuemntos hace el resto. También cuentan con un servicio para entidades comerciales.

- MkDocs es un generador de sitios estáticos rápidos y sencillo orientado a la creación de documentación de proyectos. Los archivos fuente de documentación se escriben en Markdown y se configuran con un único archivo de configuración YAML. MKDocs tiene varios temas integrados, incluido un puerto del tema de documentación Read the Docs para user con MKDocs. Uno de los temas más nuevos es MKDocs Material.

- Docusaurus es un generadr de sitios estáticos diseñado exclusivamente para crear sitios web de documentación. Admite traducciones, búsquedas y control de versiones.

- VuePress: Es un generador de sitios estáticos impulsados por Vue y optimizado para escribir documentación técnica.

- Jekyll se mencionó anteriormente en la sección de sitios web, per también

es una buena opción para generar sitio web de documentación a partir de  archivos Markdown.
