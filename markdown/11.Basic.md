## Párrafos, encabezados, citas en bloque

Un párrafo es simplemente una o más líneas de texto consecutivas separadas por una o más líneas en blanco. Los párrafos normales no deben tener sangría con espacios o tabulaciones.

Markdown ofrece dos estilos de encabezados: Setext y atx. Los encabezados de estilo Setext para "h1" y "h2" se crean "subrayando" con signos iguales (=) y giones (-), respectivamente. Para crear un encabezado estilo atx, coloque de 1 a 6 marcas de almohadilla (#) al principio de la línea; el número de almohadillas es igual al nivel de encabezado HTML resultante.

Las citas en bloque se indican mediante corchetes angulares ">@ estilo correo electrónico.

Ejemplo:

A First Level Header
===

A second Level Header
---

Now is the time for all good men to come to the aid of theri country. This is just a regular paragraph

The quick brown fox jumped over the lazy dogs back.

### Header 3

> This is a blockquote
>
> This is the second paragraph in the blockquote.
>
> ## This is ah H2 in a blockquote

## Énfasis de frase

Markdown utiliza asteriscos y guiiones bajos para indicar intervalos de énfasis

Ej:
Some of these words *are emphasized*.
Some of these words *are emphasized*.

Use two asterisks for **strong emphasis**
Of, if you prefer, **use two unsersocres instead**

### `Ejemplo: (emphasis)`

```html
<p>Algunas de estas palabras <em>se enfatizan</em>.
  Algunas de estas palabras <em>también se enfatizan</em>.
<p>Use dos asteriscos para <strong>énfasis</strong>.
  O, si lo prefiere, <strong>use dos guiones bajos</strong>.</p>
```

## Enlaces

Markdown admite dos estilos para crar enlaces: en línea y de referencia. En ambos estilos, se utilizan para delimitar el texto que se desea convertir en enlace.

Los enlaces en línea utilizan paréntesis inmediatamente después del texto del enlace

### `Ejemplo (links)`

This is an [example](http://example.com/)

```html
<p>This is an <a href="http://example.com/">example link</a>.</p>
```

## Code

Es un párrafo normal, puedes crear un fragmento de código encerrado el texto entre comillas invetidas. Los signos de ampersand (&) y los corchetes angulares (< o >) ser convertirán automáticamente en entidades HTML. Esto facilita el uso de Markdown para escribir ejemplos de código HTML.

Desaconsejo encarecidamente el uso de etiquetas `<blink>`.

Ojalá SmartyPants usara entidades con nombre como `&mdash;` en lugar de entidades codificadas en decimal como `&#8212;`.

Para especificar un bloque completo de código preformateado, identa cada línea del bloque con 4 espacios o 1 tabulación. Al igual que con los fragmentos de código, lo carecteres &, < y > se escaparán automáticamente.

Si quieres que tu página sea válida según XHTML 1.0 Strict, debes incluir etiquetas de párrafo en tus citas en bloque:

Ej:

```html
<pre>
  <code>&lt;blockquote&gt;
    &lt;p&gt;For example.&lt;/p&gt;
    &lt;/blockquote&gt;
  </code>
</pre>
```
