# Span Elements

Markdown admite dos estilos de enlaces: en línea y de referencia.

En ambos estilos, el texto del enlace se delimita con corchetes []

Para crear un enlace en línea, utilice paréntesis normales inmediatamente después del corchete de cierre del texto del enlace. Dentro de los paréntesis, escriba la URL a la que desea que apunte el enlace, junto con un título opcional para el enlace, entre comillas. Por ejemplo:

Este es un [ejemplo](http://example.com/ "Titulo") de enlace en línea.

[Este](http://example.net/) no tiene atributo.

Esto produce:

```html
<p>Este es un <a href="http://example.com/" title="Titulo"> ejemplo</a> de enlace de línea.</p>

<p><a href="http://example.net/">Este</a> no tiene atributo</p>
```

Si te refieres a un recurso local en el mismo servidor, puedes usar rutas relativas:

Puedes ver [acerca][id] enlace de estilo de referencia.

Opcionalmente, puede usar un espacio para separar los conjuntos de corchetes:

Este es [ejemplo] [id] enlace de estilo de referencia.

Luego, en cualquier parte del documento, define las etiqueta de tu enlace de esta manera, en una línea aparte:

[id]: http://example.com/  "Título opcional aquí"

Es decir:

- Corchetes que contienen el identificador del enlace (opcionalmente con sangria desde el margen izquierdo de hasta tres espacios);

- Seguido de dos puntos;

- Seguido de uno o más espacios (o tabulaciones)

- Seguido de la URL del enlace;

- Opcionalmente seguido de un atributo de título para el enalce, entre comillas dobles o simples, o entre paréntesis.

Las siguientes tres definiciones de enlace son equivalentes:

[foo]: http://example.com/  "Título opcional aquí"
[foo]: http://example.com/  'Título opcional aquí'
[foo]: http://example.com/  (Título opcional aquí)

Nota:

Existe un error conocido en Markdown.pl 1.0.1 que impide el uso de comillas simples para delimitar los títulos de los enlaces.

Opcionalmente, la URL del enalce puede ir entre corcheste angulares:

[id]: <http://example.com/>  "Título opcional aquí"

Puedes colocar el atributo de título en las siguientes líneas y usar espacios adicionales o tabulaciones para rellenar, lo que suele quedar mejor con URL más largas:

[id]: http://example.com/longish/path/to/resource/here
    "Título opcional aquí"

Las definiciones de enlace solo se utilizan para crear enlaces durante el procesamiento de Markdown y se elimina del documento en la salida HTML.

Los nombres de las definiciones de enlace pueden contener letas, números, espacios y signos de puntuación pero no distinguen entre mayúsculas y minúsucloas. Por ejemplo, estos dos enlaces, son equivalentes:

[link text][a]
[link text][A]

El atajo de nombre de enalce implícito le permite omitir el nombre del enlace, en cuyo caso el texto del enlace se utiliza como hnombre. Simplemente use un conjunto vación de corchete; por ejemplo, para enlazar la palabra "Google" con el sitio web google.com, podría escribir simplemente:

[Google][]

Y luego definir el vínculo:

[Google]: http://google.com/

Dado que los nombres de los enlaces pueden contener espacios, este atajo funciona incluso para varias palabras en el texto del enlace:

Visita [Daring Fireball][] para más información.

Las definiciones de enlaces se pueden colocar en cualquier parte de tu documento Markdown. Se pueden poner justo después de cada párrafo en el que se utilizan, pero si quieres, puedes colocarlas todas al final del documento, como si fueran notas al pie.

Aquí tienes un ejemplo de enlaces de referencia en acción:

Recibo diez veces más tráfico de [Google][1] que de [Yahoo][2] o [MSN][3].

[1]: http://google.com/ "Google"

[2]: http://search.yahoo.com/ "Búsqueda en Yahoo"

[3]: http://search.msn.com/ "Búsqueda en MSN"

La ventaja de los enlaces de estilo referencial no radica en que sean más fáciles de escribir, sino en que, con ellos, el código fuente del documento resulta mucho más legible.

Con los enlaces de estilo referencia de `Markdown`, el documento fuente se asemeja mucho más al resultado final, tal como se visualiza en un navegador. Al permitir mover los metadatos relacionados con el marcado fuera del párrafo, se pueden añadir enlaces sin interrupir el flujo narrativo del texto.
