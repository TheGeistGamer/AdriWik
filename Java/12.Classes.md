# Classes

```java
public class Main {
	public static void main(String[] args) {
		System.out.println("Hey 👋🏻👋🏻");
	}
}
```

A **class** allows you to model a real-world entity inside a program, and is the blueprint to create **object**, which are called instances of the class. Instead of writing the same code multiple times, you can define a class once and create a bunch of objects based on it.

- On `Student` class can create a bunch of `Student` objects 👩🏼‍💻👨🏻‍💻👩🏽‍💻👩‍💻🧑🏾‍💻
- On `Car` class can create a bunch of `Car` objects 🚗🚙🚕🚓
- One `Flowers` class can create a bunch of `Flowers` objects 🌷🌻🪻🌹

Here's the basic syntax for creating a class in Java:

```java
class Name {
	// Attribute inside
}
```

We can define a class for a student an provide it with attributes. These **attributes** can be both variables and methods!

```java
class Student {
	String firstName;
	String lastName;
	int sudentId;
	double gpa;
}
```

There class attributes or **instance variables** are a part of the `Student` class for use to create `Student` objects later.

The `Student` class has four attributes:

- `firstName` of the type `String`
- `lastName` of the type `String`
- `studentId` of the type `int`
- `gpa` of the type `double`

While classes are typically palced in their own files, we can include them above the `Main` class in the same **Main.java** file

```java
public class Main {
	public static void main(String[] args) {
		// Some code
	}
}
```

## Objects

An **Object** is an "instance" of a class. A class is simply a template to create objects, which are individual copies of the class with actual values.

The syntax for creating an object looks like this:

```java
Student percy = new Student();
```

Now that we have created an object of `Student` and saved it to `Percy`, we can access and edit the class attributes by using `.` dot notation

```java
class Student {
    String firstName;
    String lastName;
    int studentId;
    double gpa;
}

public class Main {
	public static void main(String[] args) {
		Student percy = new Student();
		
		percy.firstName = "Percy";
		percy.lastName = "Jackson";
		percy.studentId = 3128913723;
		percy.gpa = 2.32
	}
}
```

Thanks to the `Student` class, we've created an object that we can access its contents! Notices how when creating objects, we are back in the `main()` function

```java
System.out.println(percy.firstName);   // Output: Percy
System.out.println(percy.lastName);    // Output: Jackson
```

## Constructors

When we create an object, sometimes we want default variables declared automatically

```java
class Computer {
	String model;
	String brand;
	int serialNumber;
	double version;
}
```

A **constructor method** sets up the initial state of an object by default 

The constructor method will have the same name as its class:

```java
class Computer {
	String model;
	String brand;
	int serialNumber;
	double version;
	
	// Contructor
	Computer() {
		version = 0.0;
	}
}
```

here, we have set the default version for all computer objects
No every object created will have the version value of `0.0`

```java
public class Main {
	public static void main(String[] args) {
		Computer device = new Computer();
		System.out.println(device.version); // Output: 0.0
	}
}
```

## Parameterized Constructors

Like with normal constructor methods, the name of the constructo has the sam name as the class, but with additional parameters!

Let's say we're using the `Computer` class again, and every tiem we create a `Computer` object, we can add the model, brand, and serial number!

We can update the `Computer` class to include this new method:

```java
class Computer {
	String model;
	String brand;
	int serialNumber;
	double version;
	
	// Constructor (with parameters)
	computer(String model, String brand, int serialNumber) {
		// Keeping the default version to 0.0
		version = 0.0;
	}
}
```

### This

To assign the instance variables to parameter, we'll use the `.this` keyword inside our constructo method:

```java
Computer(String model, String brand, int serialNumber) {
	this.model = model;
	this.brand = brand;
	this.serialNumber = serialNumber;
	version = 0.0; // Not a parameter, so it doesn't need to be referenced by ".this"
}
```

When a constructo or method has parameters with the same name as its instance variable, `.this.` helps distinguish them, and assigns the parameters values to their corresponding instance variables.

