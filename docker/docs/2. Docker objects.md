# Docker objects

Al usar Docker, crear y utilizas imágenes, contenedores, redes, volúmenes, complementos y otro objetos. Esta sección ofrece una breve descripción general de algunos de esos objetos.

## Imágenes

Una imagen es una plantilla de solo lectura con instrucciones para crear un contenedor Docker. A menudo, una imagen se base en otra, con algunas personalizaciones adicionales. Por ejemplo, puedes crear una imagen basada en la imagen de Ubuntu pero que incluya el servidor web Apache y tu aplicación, así como los detalles de configuración necesarios para que tu aplicación se ejecute.

Puedes crear tus propias imágenes o usar solo las creadas por otros y publicadas en un registro. Para crear tu propio imagen, creas un DockerFile con una sitaxis sencilla que define los pasos necesarios para crearla y ejecutarla. Cada instrucción en un Dockerfile crea una capa en la imagen. Cuando modificas el Dockerfile y reconstruyes la imagen, solo se recontruyen las capas que ha cambiando. Esto es parte de lo qwue hace quel as imágenes sean tan ligeras, pequeñas y rápidas, con comparación con otras tecnologías de virtuailización.

## Contenedores

Un contenedor es una instancia ejecutable de una imagen. Puedes crear, iniciar, detener, mover o eliminar un contenedor mediante la API o la CLI de Docker. Puedes conectar un contenedor a una o más redes, adjuntarla almacenamiento o incluso crear una nueva imagen a partir de su estado actual.

Por defecto, un contenedor está relativamente bien aislaod de otros contenedores y de la máquina anfitriona. Puedes controlar el grado de aislamiento de la red, el almacenamiento y otros subsistemas subyacentes de un contenedor con respecto a otros contenedores o a la máquina anfitriona.

Un contenedor se define por su imagen y por las opciones de configuración que le proporciones al crarlo o iniciarlo. Cuando se eleimina u contenedor, cualquier cambio en su estado que no esté almacenado en el alamacenamiento persistente desaparece.

Ejemplo de comando `docker run`

El siguiente comando ejecuta un contenedor de Ubuntu, se conecta de forma interactiva a su sesión local de línea de comandos y ejecuta `/bin/bash`

```bash
docker run -i -t ubuntu /bin/bash
```

Al ejecutar este comando, sucede lo siguiente

* Si no tiene la imagen de Ubuntu localmente, Docker la descarga del registro configurado, como si hubiera ejecutado manualmente `docker pull ununtu`

* Docker crea un nuevo contenedor, como si hubera ejecutado manualmente el comando `docker container create`.

* Docker asigna un sistema de archivos de lectura y escritura al contenedor, como capa final. Esto permite que un contenedor en ejecución cree o modifique archivos y directorios en su sistema de archivos local.

* Docker crea una interfaz de red para conectar el contenedor a la red predeterminada, ya que no especificó ninguna opción de re. Esto incluye la asignación de una dirección IP al contenedor. Por defecto, los contenedores pueden conectarse a redes externas utilizando la conexión de red de la máquina anfitriona.

* Docker inicia el contenedor y ejecuta `/bin/bash`. Dado que el contenedor se ejecuta de forma interactiva y está conectado a su terminal (debido a las opciones `-i` y `-t`), puede introducir datos mediante el teclado mientras Docker registra la salida en su terminal.
Cuando ejecutas `exit` para finalizar el comando `/bin/bash`, el contenedor se detiene, pero no se elimina. Puedes reinicarlo o eliminarlo.
